Política de privacidad de GPS Logger

Última actualización: 12 de junio de 2026

GPS Logger es una aplicación de grabación privada de rutas GPS publicada por NorthforceApps. Esta política explica qué datos gestiona la aplicación, cómo se usan y qué opciones tienes.

Contacto: northforceapps [at] protonmail [dot] com

Resumen

GPS Logger graba rutas de ubicación cuando inicias una grabación, o cuando activas el inicio automático tras reiniciar y tu teléfono se reinicia. También puedes importar archivos GPX que tú elijas. Los datos de las rutas se guardan en tu dispositivo para que puedas ver tu historial, ver tu ruta en un mapa y exportar archivos GPX o KML. La aplicación no vende tus datos personales ni los comparte con terceros.

Datos que gestiona la aplicación

Datos de ubicación

Cuando inicias una grabación, GPS Logger recopila datos de ubicación precisos, como latitud, longitud, marca de tiempo, altitud, velocidad, rumbo y precisión. Cuando importas un archivo GPX, la aplicación lee los puntos de ruta de ese archivo. Estos datos se usan para crear tu historial de rutas, calcular la distancia y la velocidad, mostrar tu ruta y exportar archivos de ruta.

GPS Logger solicita acceso a la ubicación en segundo plano para que la grabación pueda continuar cuando la aplicación no está visible, la pantalla está apagada o cambias a otra aplicación. Si activas el inicio automático tras reiniciar, la ubicación en segundo plano también permite que la aplicación inicie una nueva grabación después de que tu teléfono se reinicie o se vuelva a encender. La grabación de ubicación se detiene cuando detienes la ruta.

Historial de rutas

La aplicación guarda los nombres de las rutas, las horas de inicio y fin, la duración, la distancia, el número de puntos, la velocidad máxima y los puntos de ruta relacionados en la base de datos de la aplicación, en tu dispositivo.

Ajustes de la aplicación

La aplicación guarda ajustes en tu dispositivo, como el modo de seguimiento, las preferencias de visualización, el inicio automático tras reiniciar, el destino del uso compartido en vivo, el código para compartir, la URL del servidor propio, una clave de API opcional y los ajustes de las funciones Pro y de por vida.

Adónde van tus datos de ubicación

Uso compartido en vivo opcional (App Cloud)

Si activas el uso compartido en vivo en App Cloud, GPS Logger transmite tu posición actual para que los usuarios admitidos de la aplicación que tengan tu código para compartir puedan seguirte en un mapa en vivo. Los datos transmitidos incluyen latitud, longitud, marca de tiempo, velocidad, rumbo, precisión y un identificador de cuenta anónimo.

Tu posición actual se sobrescribe en el mismo lugar mientras el uso compartido está activo: solo se almacena la posición actual en App Cloud. Un identificador de cuenta anónimo permanece asociado a tu código para compartir, con el fin de evitar que otro usuario tome el control de ese código después de que se elimine tu posición en vivo. La aplicación usa la autenticación anónima de Firebase para identificar a quien transmite y a cada instalación de la aplicación que sigue, aplicar tu límite de seguidores activos (de 1 a 20 instalaciones) y permitirte restablecer el código para revocar el acceso.

App Cloud funciona actualmente sobre Google Firebase, que actúa como nuestro proveedor de servicios para esta función. Firebase se usa únicamente para retransmitir la posición en vivo que tú decides compartir y para aplicar la propiedad del código para compartir y los límites de seguidores. No vendemos ni compartimos tus datos de ubicación con terceros.

Los datos se cifran en tránsito (HTTPS/TLS). Tu posición actual y los registros temporales de presencia de seguidores se eliminan cuando dejas de transmitir. Si restableces tu código para compartir, el registro antiguo de App Cloud se elimina y los seguidores actuales pierden el acceso de inmediato. Como medida de privacidad, cualquier posición que deje un dispositivo que no se desconecte correctamente se elimina automática y permanentemente en un plazo de 7 días. El identificador anónimo de propiedad del código para compartir permanece asociado a tu código actual para evitar usurpaciones.

Uso compartido en servidor propio opcional (tu propio servidor)

Si activas la opción de por vida «Tu propio servidor» e introduces la URL de un servidor, GPS Logger envía tu posición en vivo actual directamente a esa URL como un pequeño objeto JSON mientras el uso compartido está activo. Estos datos evitan por completo nuestra infraestructura en la nube; se dirigen exclusivamente al destino que tú indicas, y no tenemos supervisión, acceso ni responsabilidad sobre cómo ese servidor los almacena o procesa. La carga útil puede incluir un código para compartir, latitud, longitud, marca de tiempo, velocidad, rumbo y precisión. Si introduces una clave de API, se envía como una cabecera de petición X-Api-Key.

Tú eres responsable de la URL del servidor que configures. Si no configuras el uso compartido en servidor propio, GPS Logger no usa esta función.

Compras

GPS Logger usa la facturación de Google Play para las suscripciones Pro y el desbloqueo de por vida. El procesamiento de las compras lo gestiona Google Play. La aplicación recibe el estado de la compra para poder desbloquear las funciones de pago, pero no recibe los datos completos de tu tarjeta de pago.

Análisis y diagnóstico

GPS Logger incluye Firebase Analytics y Firebase Crashlytics. Estos servicios pueden recopilar identificadores de instancia de la aplicación, identificadores de dispositivo u otros identificadores, interacciones con la aplicación, eventos de compra dentro de la aplicación, una ubicación aproximada derivada de una dirección IP enmascarada, registros de fallos e información de diagnóstico. Estos datos se usan para entender el uso de la aplicación, diagnosticar fallos, mejorar el rendimiento y mantener la calidad de la aplicación.

La aplicación también puede registrar eventos como ruta iniciada, ruta detenida, ruta exportada, mejora pulsada y mejora completada. Estos eventos no incluyen de forma intencionada tus coordenadas GPS precisas.

El análisis y los informes de fallos están activados de forma predeterminada, pero son opcionales. Puedes desactivarlos en cualquier momento en Ajustes → Acerca de → «Compartir datos de uso y fallos»; al hacerlo, se desactiva en tu dispositivo la recopilación tanto de Firebase Analytics como de Crashlytics.

Mapas

GPS Logger usa componentes de Google Maps para mostrar las rutas. Google Maps y los servicios de Google Play pueden procesar los datos técnicos necesarios para ofrecer la funcionalidad de mapas, conforme a los términos y prácticas de privacidad aplicables de Google.

Cómo se usan los datos

GPS Logger usa los datos para:

  • Grabar las rutas GPS que tú inicias.
  • Mostrar las rutas actuales y guardadas en la aplicación.
  • Calcular estadísticas de la ruta, como distancia, duración, velocidad y número de puntos.
  • Importar los archivos GPX que selecciones y exportar rutas como archivos GPX o KML.
  • Ofrecer el uso compartido en vivo opcional cuando tú lo actives.
  • Ofrecer la subida a servidor propio opcional cuando tú la configures.
  • Desbloquear y verificar las compras Pro y de por vida a través de Google Play.
  • Mejorar la fiabilidad, diagnosticar fallos y entender el uso básico de la aplicación.

Cesión y divulgación

GPS Logger no vende tus datos personales ni los comparte con terceros.

Los datos solo pueden transmitirse en estos casos:

  • Decides exportar o compartir un archivo GPX o KML a través del menú para compartir de Android.
  • Activas el uso compartido en vivo de App Cloud, que retransmite tu ubicación actual a través de Firebase, nuestro proveedor de servicios, a los usuarios admitidos de la aplicación que conocen tu código para compartir.
  • Activas el uso compartido en servidor propio y facilitas la URL de un servidor.
  • Google Play procesa la información de compra para los desbloqueos Pro y de por vida.
  • Firebase Analytics y Crashlytics procesan datos de análisis, fallos, diagnóstico e identificadores como proveedores de servicios para la calidad de la aplicación.
  • Google Maps y los servicios de Google Play procesan los datos necesarios para mostrar mapas, ofrecer servicios relacionados con la ubicación y procesar compras.
  • Los datos pueden divulgarse si así lo exige la ley o para proteger derechos, la seguridad o la protección.

Almacenamiento y conservación de datos

El historial de rutas y los ajustes se guardan en tu dispositivo. Las rutas permanecen guardadas hasta que las elimines, desinstales la aplicación o borres los datos de la aplicación. Los archivos de exportación se crean en la caché de la aplicación para compartirlos y pueden ser eliminados por la aplicación o por el sistema Android.

La copia de seguridad de Android puede respaldar los ajustes de la aplicación según la configuración de tu dispositivo y de la copia de seguridad de tu cuenta de Google. GPS Logger excluye la base de datos local de rutas y la identidad del dispositivo para el uso compartido en vivo de la copia de seguridad en la nube de Android. Puedes gestionar la copia de seguridad de Android en los ajustes de tu dispositivo.

En el uso compartido en vivo de App Cloud, tu posición actual se sobrescribe a medida que te mueves; no se conserva ningún rastro del historial. Tu posición y los registros temporales de presencia de seguidores se eliminan cuando dejas de transmitir. Al restablecer tu código para compartir, se elimina el registro antiguo de App Cloud y se revoca a los seguidores actuales. Cualquier posición que quede tras una desconexión incorrecta se elimina automática y permanentemente en un plazo de 7 días. Un identificador anónimo de propiedad del código para compartir permanece almacenado para tu código actual con el fin de evitar usurpaciones.

Firebase Analytics, Crashlytics, Google Play, Google Maps y cualquier servidor propio que configures pueden tener sus propias prácticas de conservación para el procesamiento que realizan como proveedores de servicios.

Tus opciones

Puedes:

  • Iniciar y detener la grabación en cualquier momento.
  • Pausar y reanudar una grabación activa.
  • Denegar los permisos de ubicación, aunque la grabación no funcionará sin acceso a la ubicación.
  • Eliminar las rutas guardadas en la aplicación.
  • Desactivar el uso compartido en vivo.
  • Desactivar el análisis anónimo y los informes de fallos en Ajustes → Acerca de.
  • Restablecer tu código para compartir para eliminar el registro antiguo de App Cloud y revocar a los seguidores actuales.
  • Desactivar o dejar en blanco la URL del servidor para la subida a servidor propio.
  • Borrar los datos de la aplicación o desinstalarla para eliminar de tu dispositivo los datos guardados localmente.
  • Gestionar la copia de seguridad de Android y los permisos de la aplicación en los ajustes de tu dispositivo.

Privacidad de los menores

GPS Logger no está dirigido a menores. No uses la aplicación para recopilar ni compartir la ubicación de otra persona sin el permiso correspondiente.

Seguridad

GPS Logger usa el almacenamiento de aplicaciones de Android para los datos locales. Los datos transmitidos a Firebase, a los servicios de Google, a Google Play y a las URL de subida a servidor propio se cifran en tránsito mediante esos servicios o protocolos. Las versiones de lanzamiento requieren HTTPS para las URL de subida a servidor propio.

Ningún método de almacenamiento o transmisión es completamente seguro.

Cambios en esta política

Esta política puede actualizarse cuando cambien la aplicación o sus prácticas de datos. La política actualizada mostrará una nueva fecha de «Última actualización».
