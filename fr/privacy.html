Politique de confidentialité de GPS Logger

Dernière mise à jour : 12 juillet 2026

GPS Logger est une application privée d'enregistrement de traces GPS publiée par NorthforceApps. Cette politique explique quelles données l'application traite, comment elles sont utilisées et les choix que vous avez.

Contact : northforceapps [at] protonmail [dot] com

Résumé

Mise à jour du 11 juillet 2026 : Vous pouvez aussi importer une exportation Google Timeline de votre choix. Les lieux enregistrés, ainsi que les moments, notes et emoji que vous ajoutez, restent dans la base de données de votre appareil. Les zones de cartes hors ligne restent sur votre appareil jusqu’à leur suppression.

GPS Logger enregistre les traces de localisation lorsque vous lancez un enregistrement, ou lorsque vous activez le démarrage automatique après redémarrage et que votre téléphone redémarre. Vous pouvez également importer les fichiers GPX de votre choix. Les données de trace sont stockées sur votre appareil afin que vous puissiez consulter votre historique, voir votre parcours sur une carte et exporter des fichiers GPX ou KML. L'application ne vend pas vos données personnelles et ne les partage pas avec des tiers.

Données traitées par l'application

Données de localisation

Lorsque vous lancez un enregistrement, GPS Logger collecte des données de localisation précises, notamment la latitude, la longitude, l'horodatage, l'altitude, la vitesse, le relèvement et la précision. Lorsque vous importez un fichier GPX, l'application lit les points de trace de ce fichier. Ces données sont utilisées pour créer votre historique de parcours, calculer la distance et la vitesse, afficher votre itinéraire et exporter des fichiers de parcours.

GPS Logger demande l'accès à la localisation en arrière-plan pour que l'enregistrement puisse continuer quand l'application n'est pas visible, que l'écran est éteint ou que vous passez à une autre application. Si vous activez le démarrage automatique après redémarrage, la localisation en arrière-plan permet également à l'application de démarrer un nouvel enregistrement après le redémarrage de votre téléphone. L'enregistrement de la localisation s'arrête lorsque vous arrêtez le parcours.

Historique des parcours

L'application stocke les noms des parcours, les heures de début et de fin, la durée, la distance, le nombre de points, la vitesse maximale et les points de trace associés dans la base de données de l'application sur votre appareil.

Paramètres de l'application

L'application stocke les paramètres sur votre appareil, tels que le mode de suivi, les préférences d'affichage, le démarrage automatique après redémarrage, la destination du partage en direct, le code de partage, l'URL du serveur auto-hébergé, la clé API optionnelle et les paramètres des fonctionnalités Pro/à vie.

Où vont vos données de localisation

Partage en direct optionnel (App Cloud)

Si vous activez le partage en direct sur l'App Cloud, GPS Logger transmet votre position actuelle afin que les utilisateurs de l'application admis avec votre code de partage puissent vous suivre sur une carte en direct. Les données transmises comprennent la latitude, la longitude, l'horodatage, la vitesse, le relèvement, la précision et un identifiant de compte anonyme.

Votre position actuelle est écrasée sur place tant que le partage est actif — seule la position actuelle de l'App Cloud est stockée. Un identifiant de compte anonyme reste associé à votre code de partage pour empêcher un autre utilisateur de prendre le contrôle de ce code après la suppression de votre position en direct. L'application utilise l'authentification Firebase anonyme pour identifier le diffuseur et chaque installation d'application d'abonné, appliquer votre limite d'abonnés actifs (1–20 installations d'application) et vous permettre de réinitialiser le code de partage pour révoquer l'accès.

L'App Cloud fonctionne actuellement sur Google Firebase, qui agit comme notre fournisseur de services pour cette fonctionnalité. Firebase est utilisé uniquement pour relayer la position en direct que vous choisissez de partager et pour appliquer la propriété du code de partage et les limites d'abonnés. Nous ne vendons ni ne partageons vos données de localisation avec des tiers.

Les données sont cryptées en transit (HTTPS/TLS). Votre position actuelle et les enregistrements transitoires de présence des abonnés sont supprimés lorsque vous arrêtez la diffusion. Si vous réinitialisez votre code de partage, l'ancien enregistrement de partage App Cloud est supprimé et les abonnés existants perdent immédiatement l'accès. Par mesure de protection de la vie privée, toute position laissée par un appareil qui ne se déconnecte pas proprement est automatiquement et définitivement supprimée dans un délai de 7 jours. L'identifiant de propriété anonyme du code de partage reste associé à votre code actuel pour empêcher le piratage.

Partage auto-hébergé optionnel (votre propre serveur)

Si vous activez l'option à vie "Votre propre serveur" et saisissez une URL de serveur, GPS Logger envoie votre position en direct actuelle directement à cette URL sous la forme d'un petit objet JSON tant que le partage est actif. Ces données contournent entièrement notre infrastructure cloud ; elles sont acheminées uniquement vers la destination que vous fournissez, et nous n'avons aucun contrôle, accès ou responsabilité quant à la manière dont ce serveur stocke ou traite ces données. La charge utile peut inclure un code de partage, la latitude, la longitude, l'horodatage, la vitesse, le relèvement et la précision. Si vous saisissez une clé API, elle est envoyée sous forme d'en-tête de requête X-Api-Key.

Vous êtes responsable de l'URL du serveur que vous configurez. Si vous ne configurez pas le partage auto-hébergé, GPS Logger n'utilise pas cette fonctionnalité.

Achats

GPS Logger utilise Google Play Billing pour les abonnements Pro et le déblocage à vie. Le traitement des achats est géré par Google Play. L'application reçoit le statut de l'achat pour pouvoir débloquer les fonctionnalités payantes, mais elle ne reçoit pas les détails complets de votre carte de paiement.

Analyses et diagnostics

GPS Logger peut utiliser Firebase Analytics et Firebase Crashlytics pour comprendre l’utilisation de l’application, diagnostiquer les plantages, améliorer la fiabilité et maintenir la qualité de l’application.

L'application peut également enregistrer des événements tels que le démarrage, l’arrêt ou l’exportation d’un parcours, les écrans et fonctions utilisés, ainsi que les mises à niveau. GPS Logger n’envoie pas intentionnellement à ces services de coordonnées GPS, d’historique de parcours, de noms de parcours ou de lieux, de notes ni de texte de recherche.

Les analyses et les rapports de plantage sont activés par défaut mais facultatifs. Vous pouvez les désactiver à tout moment dans Paramètres → À propos → "Partager les données d'utilisation et de plantage" ; cela désactive la collecte de Firebase Analytics et Crashlytics sur votre appareil.

Cartes

GPS Logger utilise des services de cartes et de géocodage en ligne lorsque vous consultez des cartes en ligne ou demandez des noms de pays et de lieux. Ces services traitent les informations nécessaires à leur fonctionnement selon leurs propres politiques de confidentialité. Les zones de cartes hors ligne téléchargées restent sur votre appareil jusqu’à leur suppression.

Comment les données sont utilisées

GPS Logger utilise les données pour :

  • Enregistrer les traces GPS que vous lancez.
  • Afficher les parcours actuels et enregistrés dans l'application.
  • Calculer les statistiques de parcours telles que la distance, la durée, la vitesse et le nombre de points.
  • Importer les fichiers GPX que vous sélectionnez et exporter les parcours sous forme de fichiers GPX ou KML.
  • Fournir un partage en direct optionnel lorsqu'il est activé par vous.
  • Fournir un téléversement auto-hébergé optionnel lorsqu'il est configuré par vous.
  • Débloquer et vérifier les achats Pro/à vie via Google Play.
  • Améliorer la fiabilité, diagnostiquer les plantages et comprendre l'utilisation de base de l'application.

Partage et divulgation

GPS Logger ne vend pas vos données personnelles et ne les partage pas avec des tiers.

Les données ne peuvent être transmises que dans les cas suivants :

  • Vous choisissez d'exporter ou de partager un fichier GPX ou KML via la feuille de partage d'Android.
  • Vous activez le partage en direct App Cloud, qui relaye votre position actuelle via Firebase, notre fournisseur de services, aux utilisateurs de l'application admis qui connaissent votre code de partage.
  • Vous activez le partage auto-hébergé et fournissez une URL de serveur.
  • Google Play traite les informations d'achat pour les déblocages Pro/à vie.
  • Firebase Analytics et Crashlytics traitent les données d'analyse, de diagnostic et d'identifiant en tant que fournisseurs de services pour la qualité de l'application.
  • Google Maps et les services Google Play traitent les données nécessaires pour afficher des cartes, fournir des services liés à la localisation et traiter les achats.
  • Les données peuvent être divulguées si la loi l'exige ou pour protéger des droits légaux, la sécurité ou la sûreté.

Stockage et rétention des données

L'historique des parcours et les paramètres sont stockés sur votre appareil. Les parcours restent stockés jusqu'à ce que vous les supprimiez, désinstalliez l'application ou effaciez les données de l'application. Les fichiers d'exportation sont créés dans le cache de l'application pour le partage et peuvent être supprimés par l'application ou le système Android.

La sauvegarde Android peut sauvegarder les paramètres de l'application en fonction des paramètres de votre appareil et de votre compte Google. GPS Logger exclut la base de données locale des parcours et l'identité de l'appareil pour le partage en direct de la sauvegarde cloud Android. Vous pouvez gérer la sauvegarde Android dans les paramètres de votre appareil.

Pour le partage en direct App Cloud, votre position actuelle est écrasée au fur et à mesure de vos déplacements ; aucun historique n'est conservé. Votre position et les enregistrements transitoires de présence des abonnés sont supprimés lorsque vous arrêtez la diffusion. La réinitialisation de votre code de partage supprime l'ancien enregistrement de partage App Cloud et révoque les abonnés existants. Toute position laissée par une déconnexion non propre est automatiquement et définitivement supprimée dans un délai de 7 jours. Un identifiant de propriété anonyme du code de partage reste stocké pour votre code actuel afin d'empêcher le piratage du code.

Firebase Analytics, Crashlytics, Google Play, Google Maps et tout serveur auto-hébergé que vous configurez peuvent avoir leurs propres pratiques de rétention pour le traitement qu'ils effectuent en tant que fournisseurs de services.

Vos choix

Vous pouvez :

  • Démarrer et arrêter l'enregistrement à tout moment.
  • Mettre en pause et reprendre un enregistrement actif.
  • Refuser les permissions de localisation, bien que l'enregistrement ne fonctionnera pas sans accès à la localisation.
  • Supprimer les parcours enregistrés dans l'application.
  • Désactiver le partage en direct.
  • Désactiver les analyses anonymes et les rapports de plantage dans Paramètres → À propos.
  • Réinitialiser votre code de partage pour supprimer l'ancien enregistrement de partage App Cloud et révoquer les abonnés existants.
  • Désactiver ou laisser vide l'URL du serveur de téléversement auto-hébergé.
  • Effacer les données de l'application ou désinstaller l'application pour supprimer les données de l'application stockées localement sur votre appareil.
  • Gérer la sauvegarde Android et les permissions de l'application dans les paramètres de votre appareil.

Vie privée des enfants

GPS Logger ne s'adresse pas aux enfants. N'utilisez pas l'application pour collecter ou partager la position d'une autre personne sans l'autorisation appropriée.

Sécurité

GPS Logger utilise le stockage de l'application Android pour les données locales. Les données transmises à Firebase, aux services Google, à Google Play et aux URL de téléversement auto-hébergé sont cryptées en transit par ces services ou protocoles. Les versions finales nécessitent HTTPS pour les URL de téléversement auto-hébergé.

Aucune méthode de stockage ou de transmission n'est totalement sûre.

Modifications de cette politique

Cette politique peut être mise à jour lorsque l'application ou ses pratiques en matière de données changent. La politique mise à jour affichera une nouvelle date de "Dernière mise à jour".
